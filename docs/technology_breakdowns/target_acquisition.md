# Захват и наведение на цель

## Назначение

Подсистема захвата цели отвечает за реакцию БПЛА на обнаруженного
пострадавшего. После того как YOLO12s находит человека в кадре, миссионный
узел подтверждает обнаружение, разворачивает дрон на цель, выполняет
визуальный подлет и затем передает управление этапу эвакуации.

Основная логика находится в:

- `src/offboard_takeoff/offboard_takeoff/node.py`
- `src/offboard_takeoff/offboard_takeoff/yolo_detector.py`

## Входные данные от YOLO

Детектор публикует лучший bounding box человека в topic:

```text
yolo/target_bbox
```

Формат сообщения `std_msgs/Int32MultiArray`:

```text
[confidence_milli, center_x, center_y, width, height, image_width, image_height]
```

Где:

- `confidence_milli` - уверенность детектора, умноженная на 1000;
- `center_x`, `center_y` - центр bounding box в пикселях;
- `width`, `height` - размер bounding box в пикселях;
- `image_width`, `image_height` - размер кадра.

В `node.py` эти данные превращаются в объект `PersonTarget`.

## Ошибка наведения

Для наведения используется горизонтальная ошибка центра bbox:

```text
center_x_error = (bbox_center_x - image_width / 2) / (image_width / 2)
```

Если значение близко к нулю, человек находится около центра кадра. Если
значение отрицательное или положительное, цель смещена влево или вправо.

Ошибка сглаживается параметром:

```text
follow_error_filter_alpha = 0.25
```

Сглаживание уменьшает дерганье из-за нестабильного bbox.

## Подтверждение цели

Дрон не реагирует на единичное срабатывание детектора. Перед переходом к
подлету цель должна удерживаться в кадре:

```text
follow_detection_confirm_sec = 1.0
```

Если bbox пропадает ненадолго, подтверждение не сбрасывается:

```text
follow_detection_gap_tolerance_sec = 0.5
```

Если пропадание длится дольше, подтверждение начинается заново.

Свежесть bbox контролируется параметром:

```text
follow_person_timeout_sec = 1.0
```

Если данные от YOLO старше 1 секунды, цель считается потерянной.

## Состояния наведения

После подтверждения цели миссия переходит в состояние:

```text
FOLLOW_PERSON
```

Внутри него используется последовательность фаз:

```text
STOP_BEFORE_APPROACH
ALIGN_TO_TARGET
APPROACH_FORWARD
HOLD_AFTER_APPROACH
```

## STOP_BEFORE_APPROACH

На этой фазе дрон зависает перед началом подлета:

```text
follow_stop_before_approach_sec = 1.0
```

Это нужно, чтобы стабилизировать изображение и запомнить начальный размер
bbox человека.

## ALIGN_TO_TARGET

Дрон поворачивает yaw в сторону цели. Управление строится по горизонтальной
ошибке bbox:

```text
yaw_rate_cmd = follow_yaw_kp * center_x_error
```

Основные параметры:

```text
follow_yaw_kp = 0.9
max_yaw_rate = 0.4
follow_yaw_deadband = 0.05
```

Если ошибка становится меньше `follow_yaw_deadband`, цель считается
выровненной по центру кадра, и дрон переходит к движению вперед.

Если bbox временно пропадает во время выравнивания, используется память
последней значимой ошибки:

```text
follow_target_memory_activation_sec = 1.0
follow_target_memory_timeout_sec = 1.0
follow_align_dropout_continue_sec = 1.0
```

Это позволяет продолжить поворот короткое время даже при кратковременной
потере bbox.

## APPROACH_FORWARD

После выравнивания дрон движется вперед по текущему курсу:

```text
follow_forward_speed = 0.35
```

Дальность до человека напрямую не измеряется. Вместо этого используется рост
bounding box в изображении. При входе в `FOLLOW_PERSON` запоминается начальная
высота bbox:

```text
person_reference_bbox_height
```

Дрон летит вперед, пока текущая высота bbox не станет больше начальной в
заданное число раз:

```text
follow_bbox_growth_target = 4.0
```

Условие завершения подлета:

```text
current_bbox_height >= person_reference_bbox_height * 4.0
```

Если bbox временно пропал во время движения вперед, дрон продолжает движение.
Остановка происходит только после появления свежего bbox, который подтвердит
достаточный рост размера цели в кадре.

## HOLD_AFTER_APPROACH

Когда bbox достиг нужного размера, дрон зависает:

```text
follow_hover_after_approach_sec = 5.0
```

После этого действие с человеком считается завершенным, и миссия переходит к
этапу эвакуации.

## Общий таймаут

Для защиты от зависания в режиме наведения используется общий таймаут:

```text
follow_person_action_timeout_sec = 90.0
```

Если `FOLLOW_PERSON` длится дольше этого времени, миссия считает ситуацию
ошибочной и переходит к восстановлению: возврат домой или failsafe.

## Этап после захвата цели

После завершения `HOLD_AFTER_APPROACH` вызывается функция завершения действия
с человеком. Далее:

1. строится маршрут эвакуации A*;
2. публикуется маршрут для карты сверху;
3. дрон переходит в `RETURN_HOME`;
4. если A* маршрут найден, возврат идет по waypoint-ам маршрута;
5. если маршрут не найден, используется прямой возврат домой.

## Краткое описание для диплома

Наведение на пострадавшего выполняется по данным bounding box от YOLO12s.
После обнаружения цель должна удерживаться в кадре в течение заданного
времени, что снижает вероятность реакции на ложные срабатывания. Затем БПЛА
стабилизируется, выравнивает курс по горизонтальной ошибке центра bounding
box и выполняет поступательное движение вперед. Оценка приближения выполняется
по росту высоты bounding box в изображении: когда bbox увеличивается в
заданное число раз относительно начального размера, БПЛА считает, что достиг
зоны взаимодействия, зависает и переходит к этапу эвакуации.
